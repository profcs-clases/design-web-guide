01

HTML Semántico

Estructura con significado

HTML semántico significa usar las etiquetas correctas para el contenido correcto. No es solo para "verse bien" — afecta accesibilidad, SEO y mantenimiento.

¿Por qué importa?

Un lector de pantalla puede decir "navegación principal" en lugar de "grupo de 5 enlaces" si usas <nav>. Esto cambia la experiencia para usuarios con discapacidad visual.

Antes vs Después

Mal: Todo son divs

<div class="header">...</div>
<div class="menu">...</div>
<div class="main">
  <div class="article">...</div>
  <div class="sidebar">...</div>
</div>
<div class="footer">...</div>

Bien: Semántico

<header>...</header>
<nav>...</nav>
<main>
  <article>...</article>
  <aside>...</aside>
</main>
<footer>...</footer>

Etiquetas clave

Etiqueta Uso Ejemplo
<header> Cabecera de página o sección Logo + navegación
<nav> Navegación principal Menú de secciones
<main> Contenido principal (único) Contenido de la página
<section> Sección temática Capa del formulario
<article> Contenido independiente Una tarjeta de estudiante
<aside> Contenido relacionado Filtros laterales
<footer> Pie de página Copyright, enlaces legales

Estructura base recomendada

HTML 
<!DOCTYPE html>
<html lang="es">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi Sistema</title>
</head>
<body>
    <header>
        <nav aria-label="Principal">
            <ul>
                <li><a href="/">Inicio</a></li>
                <li><a href="/registro">Registros</a></li>
            </ul>
        </nav>
    </header>
    
    <main>
        <section aria-labelledby="titulo-seccion">
            <h2 id="titulo-seccion">Listado</h2>
            <article>
                <!-- Contenido -->
            </article>
        </section>
        
        <aside aria-label="Filtros">
            <!-- Filtros laterales -->
        </aside>
    </main>
    
    <footer>
        <p>&copy; 2025 Mi Sistema</p>
    </footer>
</body>
</html>
Jerarquía de encabezados: Usa un solo <h1> por página, luego <h2>, <h3>, etc. en orden lógico. No saltes niveles (h2 → h4 está mal).

Reto rápido

Abre el código fuente de tu página web favorita. ¿Usa HTML semántico o todo son <div>? Identifica 3 etiquetas semánticas que usa correctamente y 3 lugares donde podría mejorar.

02

Formularios: La puerta de entrada

Donde los usuarios interactúan con tu sistema

Los formularios son el componente más crítico en sistemas administrativos. Un mal formulario = datos malos = sistema inútil.

Principios de formularios efectivos

Minimiza campos

Pide solo lo estrictamente necesario

Agrupa lógicamente

Usa fieldset para secciones relacionadas

Valida en tiempo real

Feedback inmediato mientras escribe

Sé explícito

Labels claros, placeholders útiles

Formulario bien estructurado

HTML 
<form action="/registro" method="POST" novalidate>
    <fieldset>
        <legend>Datos personales</legend>
        
        <div class="form-group">
            <label for="nombre">Nombre completo *</label>
            <input 
                type="text" 
                id="nombre" 
                name="nombre"
                required
                minlength="3"
                maxlength="100"
                autocomplete="name"
                aria-describedby="nombre-help"
            >
            <small id="nombre-help">Como aparece en tu cédula</small>
            <span class="error-message" role="alert"></span>
        </div>

        <div class="form-group">
            <label for="cedula">Cédula *</label>
            <input 
                type="text" 
                id="cedula" 
                name="cedula"
                required
                pattern="V-\d{7,8}|E-\d{7,8}"
                placeholder="V-12345678"
                inputmode="numeric"
            >
        </div>
    </fieldset>

    <fieldset>
        <legend>Datos académicos</legend>
        
        <div class="form-group">
            <label for="anio">Año a cursar *</label>
            <select id="anio" name="anio" required>
                <option value="">Seleccione...</option>
                <option value="1">1er año</option>
                <option value="2">2do año</option>
                <option value="3">3er año</option>
            </select>
        </div>
    </fieldset>

    <button type="submit" class="btn btn-primary">
        Registrar
    </button>
</form>

Tipos de input clave

Tipo Uso Ventaja
email Correos electrónicos Validación automática + teclado optimizado en móvil
tel Teléfonos Teclado numérico en móvil
number Cantidades, notas Controles nativos + min/max
date Fechas DatePicker nativo del navegador
url Sitios web Validación + tecla .com en móvil
search Búsquedas Botón de limpiar nativo

Validación nativa HTML5

HTML 
<!-- Atributos de validación -->
<input type="text" required>                    <!-- Obligatorio -->
<input type="text" minlength="3">               <!-- Mínimo caracteres -->
<input type="text" maxlength="50">              <!-- Máximo caracteres -->
<input type="number" min="0" max="20">          <!-- Rango numérico -->
<input type="text" pattern="[A-Z]{3}-\d{4}">    <!-- Expresión regular -->
<input type="email">                            <!-- Validación email -->
<input type="url">                              <!-- Validación URL -->
Siempre valida en el backend también. La validación del frontend es para UX; la del backend es para seguridad. Nunca confíes solo en el cliente.

Atributos de accesibilidad clave

Atributo Propósito
label[for] Asocia una etiqueta con un input
aria-describedby Referencia un texto descriptivo
aria-invalid Indica si el valor es inválido
required Marca el campo como obligatorio
autocomplete Sugiere valores al navegador

Reto rápido

Crea un formulario de 5 campos usando solo HTML. Debe tener: validación nativa, labels asociados, agrupación con fieldset y mensajes de ayuda. Pruébalo sin JavaScript.

03

CSS Moderno para Interfaces

Las herramientas del siglo XXI

Variables CSS (Custom Properties)

Las variables CSS te permiten definir valores reutilizables. Son la base del modo oscuro y de sistemas de diseño mantenibles.

CSS 
:root {
    /* Colores */
    --color-primary: #3b82f6;
    --color-success: #10b981;
    --color-warning: #f59e0b;
    --color-danger: #ef4444;
    
    /* Neutros */
    --color-bg: #ffffff;
    --color-surface: #f9fafb;
    --color-text: #111827;
    --color-text-muted: #6b7280;
    --color-border: #e5e7eb;
    
    /* Espaciado */
    --space-xs: 0.25rem;   /* 4px */
    --space-sm: 0.5rem;    /* 8px */
    --space-md: 1rem;      /* 16px */
    --space-lg: 1.5rem;    /* 24px */
    --space-xl: 2rem;      /* 32px */
    
    /* Tipografía */
    --font-sans: 'Inter', system-ui, sans-serif;
    --font-mono: 'Fira Code', monospace;
    
    /* Bordes */
    --radius-sm: 4px;
    --radius-md: 8px;
    --radius-lg: 12px;
    
    /* Sombras */
    --shadow-sm: 0 1px 2px rgba(0,0,0,0.05);
    --shadow-md: 0 4px 6px rgba(0,0,0,0.1);
    --shadow-lg: 0 10px 15px rgba(0,0,0,0.1);
}

Flexbox vs Grid: ¿Cuándo usar cada uno?

Flexbox

Unidimensional (fila O columna)

  • Menús de navegación
  • Alinear elementos
  • Distribuir espacio
  • Centrar contenido

Grid

Bidimensional (filas Y columnas)

  • Layouts de página
  • Galerías
  • Dashboard
  • Formularios complejos

Grid: Layout de dashboard

CSS 
.dashboard {
    display: grid;
    grid-template-columns: 250px 1fr;
    grid-template-rows: 60px 1fr;
    grid-template-areas:
        "sidebar header"
        "sidebar main";
    min-height: 100vh;
}

.sidebar { grid-area: sidebar; }
.header  { grid-area: header; }
.main    { grid-area: main; }

/* Responsive: colapsa en móvil */
@media (max-width: 768px) {
    .dashboard {
        grid-template-columns: 1fr;
        grid-template-areas:
            "header"
            "main";
    }
    .sidebar { display: none; }
}

Flexbox: Tarjetas de resumen

CSS 
.stats-grid {
    display: flex;
    flex-wrap: wrap;
    gap: var(--space-md);
}

.stat-card {
    flex: 1 1 200px; /* Crece, se encoge, base 200px */
    background: var(--color-surface);
    padding: var(--space-lg);
    border-radius: var(--radius-md);
}

/* En móvil se apilan automáticamente */

Transiciones y micro-interacciones

CSS 
.btn {
    transition: all 0.2s ease;
}

.btn:hover {
    transform: translateY(-2px);
    box-shadow: var(--shadow-md);
}

.btn:active {
    transform: translateY(0);
}

/* Animación de entrada */
@keyframes fadeInUp {
    from {
        opacity: 0;
        transform: translateY(20px);
    }
    to {
        opacity: 1;
        transform: translateY(0);
    }
}

.card {
    animation: fadeInUp 0.3s ease-out;
}

Frameworks CSS: ¿cuál elegir?

Bootstrap

El clásico

Sistema de grid, componentes pre-diseñados, ideal para prototipar rápido.

~170KB con JS

Tailwind CSS

Utility-first

Clases utilitarias, altamente personalizable. Curva alta pero muy flexible.

~10KB en producción

CSS Puro

Control total

Sin dependencias, rendimiento óptimo. Ideal para aprender profundamente.

0KB (solo tu código)
Recomendación: Si estás empezando, usa CSS puro para entender los fundamentos. Luego prueba Bootstrap para prototipar rápido. Tailwind es excelente cuando ya dominas CSS.

Reto rápido

Define un set de variables CSS para un tema "institucional" (azules, grises). Luego crea un grid de 3 columnas que se convierta en 1 columna en móvil.

04

Diseño Responsive

Una interfaz, múltiples dispositivos

Más del 60% del tráfico web viene de móviles. En Venezuela, con conexiones lentas y dispositivos antiguos, el diseño responsive no es opcional.

Mobile-first vs Desktop-first

Mobile-First ✓

Diseñas para móvil primero, luego agregas complejidad.

.grid { 
    grid-template-columns: 1fr;
}
@media (min-width: 768px) {
    .grid { grid-template-columns: 1fr 1fr; }
}
@media (min-width: 1024px) {
    .grid { grid-template-columns: repeat(3, 1fr); }
}

Desktop-First ✗

Diseñas para desktop, luego "arreglas" para móvil.

.grid {
    grid-template-columns: repeat(3, 1fr);
}
@media (max-width: 1024px) {
    .grid { grid-template-columns: 1fr 1fr; }
}
@media (max-width: 768px) {
    .grid { grid-template-columns: 1fr; }
}

Breakpoints recomendados

Nombre Tamaño Dispositivo típico
xs< 576pxMóvil vertical
sm≥ 576pxMóvil horizontal
md≥ 768pxTablet
lg≥ 992pxLaptop
xl≥ 1200pxDesktop
xxl≥ 1400pxDesktop grande

Unidades relativas

Unidad Relativa a Uso recomendado

%PadreAnchos de contenedores
vw/vhViewportSecciones full-screen

Viewport meta tag: imprescindible

HTML 
<!-- ESTA LÍNEA ES OBLIGATORIA en el <head> -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">
Prueba siempre en dispositivo real. Las devtools del navegador son útiles, pero no reemplazan probar en un móvil real con conexión lenta. En Venezuela, prueba con 3G.

Reto rápido

Crea una tarjeta de producto que sea:
• En móvil: imagen arriba, texto debajo
• En tablet: imagen izquierda, texto derecha
• En desktop: igual que tablet pero con más padding
